# Task 6 REST API Design

For this task, I used the design-first approach to create the API for the Meeting Scheduler.

I started by writing the api.yaml file using the OpenAPI 3.0 standard. This file defines all the endpoints, data models, and rules for the API.

Once the YAML specification was finished, I generated the documentation.html file using Swagger Editor. This HTML file provides a readable version of the documentation.

The api.yaml is the main source file, and the documentation.html is for viewing the documentation.

The actual server implementation of this design is done in Task 7.
